Sous-systeme "Archivage"

В конец документа

Данный режим работы предназначен для ведения архивов баз данных. Ведение архивной информации необходимо для хранения и аварийного восстановления баз данных. Внешний вид окна подсистемы "Архивы БД" представлен на рисунке 1.



Рисунок 1 - Архивы БД по умолчанию

В данной подсистеме возможны следующие действия:

  1. Заархивировать БД в папке по умолчанию. Для этого сделайте следующие действия:
    1. Выбираем - "Заархивировать базы данных", смотри рисунок 1.
    2. Ставим галочку "Архивы в рабочем каталоге", смотри рисунок 1.
    3. Нажать кнопку "Выполнить."
    4. Дождаться сообщения заархивировано. смотри рисунок 2.


      Рисунок 2 - Сообщение "Заархивировано"
    5. Нажать кнопку "OK". После нажатии данной кнопки произойдет возврат к главной форме подсистемы "Архивы БД".
  2. Заархивировать БД в выбранной папке. Для этого сделайте следующие действия:
    1. Выбираем - "Заархивировать базы данных", смотри рисунок 3.
    2. Убираем галочку "Архивы в рабочем каталоге", смотри рисунок 3.


      Рисунок 3 - Архивы БД в выбранной папке
    3. Нажать кнопку "Выполнить.". После этого откроется окно смотри рисунок 4.


      Рисунок 4 - Выбор папки сохранения архива
      1. Выбрать папку в которую будет сохранятся архив.
      2. Ввести имя архива.
      3. Нажать кнопку "OK".
      4. Дождаться сообщения "Заархивировано". смотри рисунок 2.
      5. Нажать кнопку "OK". После нажатии данной кнопки произойдет возврат к главной форме подсистемы "Архивы БД".
  3. Извлечь архив БД в папку по умолчанию. Для этого сделайте следующее действия:
    1. Выбираем - "Извлечь базы данных из архива", смотри рисунок 5.
    2. Ставим галочку "Архивы в рабочем каталоге", смотри рисунок 5.


      Рисунок 5 - Извлечение архива БД по умолчанию
    3. Нажать кнопку "Выполнить."
    4. Дождаться сообщения "Распаковано". смотри рисунок 6.


      Рисунок 6 - Сообщение "Распаковано"
    5. Нажать кнопку "OK". После нажатии данной кнопки произойдет возврат к главной форме подсистемы "Архивы БД".
  4. Извлечь архив БД в выбранную папку. Для этого сделайте следующее:
    1. Выбираем - "Извлечь базы данных из архива", смотри рисунок 7.
    2. Убираем галочку "Архивы в рабочем каталоге", смотри рисунок 7.


      Рисунок 7 - Извлечение архива БД в выбранную папку
    3. Нажать кнопку "Выполнить."После этого откроется окно смотри рисунок 8.


      Рисунок 8 - Выбор файла архива
      1. Выбрать файл архива *.rar.
        1. При создании архива ничего не меняли:
          • Заходим в папку программы(по умолчанию C:\Program Files (x86)\ASPU).
          • Заходим в папку - Archive.
          • Выбираем файл - АСПУ.rar.
          • Нажимаем кнопку "OK".
        2. При создании архива изменили путь сохранения архива или имя архива.
          • Находим папку в которую сохранили архив.
          • Выбираем файл архива *.rar или *.RAR (*-имя которым назвали архив).
          • Нажимаем кнопку "OK".
      2. Выбираем папку в которую архив извлекается, смотри рисунок 9.


        Рисунок 9 - Выбор папки извлечения архива
      3. Нажать кнопку Select.
      4. Дождаться сообщения - "Распаковано" смотри рисунок 6.
      5. Нажать кнопку "OK". После нажатии данной кнопки произойдет возврат к главной форме подсистемы "Архивы БД".

Примечание: для того чтобы восстановить бд из извлеченного архива, надо проделать следующие действия:

  1. Зайти в папку в которую извлекли архивы(По умолчанию: C:\SGA\Архивы БД).
  2. Выделить все файлы *.dbf, *.cdx, *.fpt(выделять - ctrl+A или ctrl+левая кнопка мыши).
  3. Нажать правой кнопкой мыши, для открытия подменю.
  4. Из выпадшего подменю нажать кнопку "Копировать".
  5. Зайти в папку с программой
  6. Нажать правой кнопкой мыши на свободном месте.
  7. Из выпавшего подменю нажать вставить, если система спрашивает заменить ли файлы, смотри рисунок 10, ответить - "Копировать с заменой".


    Рисунок 10 - Вопрос о копировании с заменой

CTRL+C - Копировать, CTRL+V - вставить.
